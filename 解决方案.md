完整的用户流程图 
清晰地展示了用户的每一步操作和状态变化。
阶段一：【生态入口】初次接触与轻量注册（暂不需要自己设计）
- 1.1 [外部] 访问官网：
  - 用户访问“奇绩创坛”官方网站。
- 1.2 [官网] 轻量注册：
  - 用户在官网点击[加入社区]，完成“轻注册”（手机号/邮箱、姓名、邀请码）。
- 1.3 [首次进入] 登陆“奇绩潜空间”门户：
  - 注册成功后，用户首次被引导至“奇绩潜空间”主页。
  - 用户状态： 访客 (Guest) ，但不是匹配功能激活用户 (Matcher)。

阶段二：【门户探索】（奇迹潜空间主页面）
- 2.1 [门户] 浏览内容：
  - 用户在“奇绩潜空间”主页，可以自由浏览“历史活动”、“最近消息”、“知识库”等高质量内容。
  - 目标： 让用户感受到社区的价值，建立信任。
- 2.2 [门户] 遇见“Co-founder Match”入口 (关键引流):
  - 用户在页面下方，看到一个名为“名片池”的专属模块。
  - 核心设计 (你的“解锁”逻辑):
    - 这个模块会展示3-4张模糊的、像素化的、不可点击的“匿名卡片”。
    - 卡片下方有一句引人入胜的文案：“超过100位顶尖的技术专家与商业精英正在此寻找伙伴。完善你的Profile，解锁并加入他们。”
    - 模块中央是一个非常醒目的按钮：[ 完善Profile，进入引力场 ]。

阶段三：【激活身份】（Profile的强制完善）
- 3.1 [触发] 点击解锁入口：
  - 用户点击了[ 完善Profile，进入引力场 ]按钮。
  - 另一个触发点： 用户点击页面右上角自己的头像，选择“完善我的Profile”。
- 3.2 [拦截] 进入“Profile引导流程”:
  - 系统检测到该用户的Profile完整度不足（例如，<90%），强制将用户带入我们之前设计的、分步骤的“Onboarding引导流程”。
  - 用户必须依次完成【我是谁】、【我的能力】、【我在寻找】等所有必填项。
- 3.3 [完成] 身份激活：
  - 用户完成引导流程，点击最后的“完成”按钮。
  - 用户状态： 从社区成员 升级为 匹配功能激活用户 (Matcher)。

阶段四：【核心应用】进入名片池并开始匹配（Co-founder Matching主页面）
- 4.1 [无缝跳转] 首次进入“名片池”:
  - Profile完善后，系统自动将用户重定向到“Co-founder Matching应用”的“名片池”页面（此页面也是主页面）。
  - 体验： 用户会感受到一种“解锁新世界”的即时满足感。他看到的，是清晰的、可交互的、真实的匿名卡片。
- 4.2 [应用内] 核心匹配循环：
  - 探索： 用户在“名片池”通过标签筛选，对感兴趣的匿名卡片点击 [我想聊聊]。
  - 响应： 用户在“通知中心”处理他人发来的兴趣信号，点击 [🤝 想聊聊] 或 [👋 暂不考虑]。
  - 连接： 双方互表兴趣后，“连接”建立，身份解锁。
  - 管理： 用户在“已连接”页面，管理自己的人脉。
- 4.3 [应用内] 返回门户：
  - 在“Co-founder Matching应用”的导航栏中，始终会有一个清晰的[ 返回潜空间门户 ]的链接，确保用户可以在两个空间之间顺畅切换。

阶段五：【价值闭环】反馈与持续优化
- 5.1 [系统] 7天后回访：
  - 对于每一个成功的“连接”，系统在7天后自动发起反馈问卷。
- 5.2 [运营] 数据驱动优化：
  - 收集到的数据用于优化社区和未来推荐逻辑。

【奇绩潜空间门户】模块
1. 模块定位与核心目标 
- 定位： “奇绩潜空间”门户是用户在“轻注册”后进入的第一个空间。是一个“价值展厅”。
- 核心目标：
  1. 建立信任： 通过展示奇绩创坛的专业内容和社区活动，快速建立用户对平台的信任感和专业度认知。
  2. 提供价值： 让用户即使不使用匹配功能，也能在此获得行业洞察和学习资源，从而提升用户粘性。
  3. 高效引流： 作为通往核心产品“Co-founder Matching应用”的最重要、最具吸引力的入口，激发用户完成Profile的强烈动机。

3. 页面模块详述 (Module Specifications)
3.1 顶部导航栏 
- 功能描述： 提供最基础的导航和用户身份展示。
- 页面元素：
  - 左侧： “奇绩潜空间” Logo。
  - 右侧： 用户头像（初期可以是灰色默认头像）和姓名（轻注册时填写的）。
- 交互逻辑：
  - 【关键】 点击用户头像/姓名，会弹出一个下拉菜单，包含两个选项：
    1. [ 完善我的Profile ] -> 点击后，触发我们设计的“强制Profile引导流程”。
    2. [ 退出登录 ]
3.2 精选内容模块（中间位置）
- 功能描述： 页面的“黄金位置”，用于展示最新、最重要、最吸引眼球的内容。
- 页面元素：
  - 一个横向轮播图
  - 内容示例：
    - “本周必读：陆奇博士关于AI Agent的最新思考”
    - “活动预告：下一期奇绩潜空间‘AI+生命科学’专场”
    - “深度报告：AIGC行业应用趋势分析”
- 数据来源 (MVP)： 由运营团队在后台手动配置。每周更新1-3篇最核心的内容即可。
- 交互逻辑： 点击卡片，可以跳转到文章详情页或活动报名页。
3.3 历史活动模块 （左侧位置）
- 功能描述： 以时间线的形式，展示“奇绩潜空间”过往的精彩活动，建立社区的厚重感和历史感。
- 页面元素：
  - 一个垂直列表，每一项代表一次过往活动。
  - 每项包含： 活动主题、举办日期、核心嘉宾、一张现场照片的缩略图。
- 数据来源 (MVP)： 一个静态列表，每次活动结束后由运营手动添加。
- 交互逻辑： 点击某一项，可以查看本次活动的内容回顾、纪要或视频录像。
3.4 知识库模块 (右侧位置)
- 功能描述： 提供有深度价值的内容，是用户反复访问的理由。
- 页面元素：
  - 分为几个固定的类别，如 [ 创业必读 ] [ 前沿技术 ] [ 行业报告 ]。
  - 每个类别下，是精选的文章或资源链接列表。
- 数据来源 (MVP)： 一个由运营团队手动维护的精选资源列表。
3.5 Co-founder Match引流模块（下部位置）
- 功能描述： 这是整个门户页面的“转化引擎”，其设计的唯一目的，就是激发用户点击并完成Profile。
- 页面元素：
  - 模块标题 (大字号，加粗): “寻找你的联合创始人”
  - 副标题/引言: “超过100位顶尖的技术专家、产品经理与商业精英正在引力场中寻找下一个机会。他们或许正在等你。”
  - 视觉核心 (模糊卡片区):
    - 横向排列3-4张经过毛玻璃或像素化效果处理的“匿名卡片”。
    - 用户隐约能看到卡片上有头像轮廓、文字和标签，但无法看清具体内容。
    - 当鼠标悬浮在模糊卡片上时，可以出现“完善Profile后即可查看”的提示。
  - 行动召唤按钮:
    - 在模糊卡片下方，放置一个全页面最醒目的、最大尺寸的按钮。
    - 按钮文案： [ ✨ 完善Profile，进入引力场 ]
- 交互逻辑：
  - 点击此按钮，将触发“阶段三：激活身份”的用户流程。
  - 系统会判断用户的Profile是否完整。如果不完整，则强制进入分步骤的Onboarding引导流程。如果用户已经完成了Profile，则直接无缝跳转到“Co-founder Matching应用”的“名片池”页面。

1. 模块总览 
- 定位： 本模块是“奇绩引力”产品的核心功能区，是一个独立的、专注的匹配应用。
- 四大核心页面： ✨ 名片池 🔔 通知中心 🤝 已连接 👤 个人Profile。

  3. 发起连接： 用户点击 [我想聊聊] 按钮，向对方发送一个单向、私密的兴趣信号。

3. 🔔 通知中心
- 核心目标： 成为用户的“社交决策中心”。用户在此处理收到的所有兴趣信号，决定是否要开启一段新的连接。
- 页面布局：
  - 采用简洁的列表布局，每条通知占一行，按时间倒序排列。
  - 未读通知有视觉提示（如红色圆点）。
- 核心元素 (每条通知)：
  - 左侧 (信息摘要): (👤) 匿名头像 + “一位 [对方的Title] 对你感兴趣”。
  - 右侧 (决策按钮组): [🤝 我想聊聊] [👋 暂不考虑]。
- 交互逻辑与承接：
  1. 深入了解： 用户点击摘要文本区域，会弹出一个模态窗口(Modal)，展示对方更详细的匿名Profile，帮助用户在不离开当前页面的情况下做出更明智的判断。
  2. 同意连接： 用户点击 [🤝 我想聊聊]。
    - 系统判断： 这是关键的“双向确认”环节。系统会记录用户的同意。
    - 触发“连接”： 系统立即执行“连接建立”的自动化流程（发送恭喜通知、解锁双方身份、在“已连接”页面中添加记录）。
    - 界面反馈： 当前这条通知的操作按钮会变为一个绿色的标签 [ ✅ 已连接 ]，并将在稍后自动归档。
  3. 礼貌拒绝： 用户点击 [👋 暂不考虑]。
    - 界面反馈： 该条通知会立即从列表中移除或变灰，给用户一种“任务已完成”的清爽感。

    1. 通知按时间倒序排列，最新的在最上方。

    3. 每条通知包含：
      1. 摘要信息： “一位 [对方Title] 对你感兴趣”。

      1. 系统记录“用户B对用户A表达了兴趣”。
      2. 系统判定为“双向匹配成功”。
      3. 系统将A和B的关系标记为“已连接”。
      4. 系统向A和B双方同时发送一条“连接成功”的通知（例如，邮件+站内信）（自动化工作流完成）

      1. 该条通知的操作按钮组变为一个绿色的标签 [ ✅ 已连接 ]。

  4. 交互动作2：点击 [👋 暂不考虑]
    1. 触发点： 用户B点击了A发来通知的 [👋 暂不考虑] 按钮。

      1. 系统记录“用户B拒绝了用户A的连接请求”。

    2. 前端展示： 弹出一个模态窗口，展示对方更详细的匿名Profile，方便用户在做决定前获取更多信息。

- 当且仅当，你也对那个向你表达兴趣的人，点击了[🤝 想聊聊]之后（或者你在“引力场”主动感兴趣的人，恰好也对你感兴趣），“连接”才会正式建立。

- 在连接建立7天后，系统会自动向双方发送一份简短的问卷，询问交流是否发生，以及体验如何。
- 收集到的所有反馈，都将用来优化我们的社区氛围和未来的匹配推荐逻辑。用户的每一次反馈，都在让这个“探索页面”变得更强。
