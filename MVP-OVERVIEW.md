# Co-founder Matching MVP - 项目概括

## 🎯 项目定位

**产品名称**: 奇绩引力场 - Co-founder Matching Platform (MVP版本)

**一句话描述**: 为创业者提供基于双向匹配的联合创始人配对平台，通过匿名探索+实名连接机制，实现高质量的合伙人匹配。

## 🚀 MVP核心价值主张

1. **匿名探索**: 用户在不暴露真实身份的情况下浏览潜在合伙人
2. **双向确认**: 只有双方互相感兴趣才能建立连接，避免骚扰
3. **标签驱动**: 通过能力标签和寻找方向快速找到匹配的人
4. **身份解锁**: 匹配成功后自动解锁真实身份信息

## 📊 MVP功能范围

### ✅ 包含功能
- 用户注册登录（邮箱+密码）
- 3步快速Profile填写（基本信息→能力标签→寻找方向）
- 名片池浏览（匿名卡片展示）
- 标签筛选（按能力/方向筛选用户）
- 发起兴趣（"我想聊聊"按钮）
- 双向匹配（自动检测+建立连接）
- 已连接列表（查看匹配成功的用户）
- Profile查看（基于关系的权限控制）

### ❌ 暂不包含（后续迭代）
- 门户引流页面
- 实时通知中心（用邮件代替）
- 复杂隐私控制（使用默认规则）
- 头像上传（用默认头像）
- 每日发起次数限制
- 7天反馈问卷
- 管理后台

## 🎨 核心用户流程

```
注册登录 → 填写Profile (3步) → 进入名片池 → 浏览+筛选
         ↓
    点击"我想聊聊"
         ↓
    系统检测双向匹配
         ↓
    匹配成功 → 解锁身份 → 查看完整Profile → 线下联系
```

## 👥 目标用户

1. **连续创业者**: 有创业经验，寻找新项目合伙人
2. **技术专家**: 有技术能力，想找商业合伙人
3. **产品经理**: 有产品思维，想找技术+商业合伙人
4. **投资人/资源方**: 有资源，想找执行团队

## 📈 关键成功指标 (KPIs)

1. **注册用户数**: 目标50+（第一个月）
2. **Profile完成率**: 目标80%+
3. **匹配成功率**: 目标20%+
4. **连接后交流率**: 目标50%+

## 🛠 技术选型概要

- **前端**: Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui
- **后端**: Supabase (PostgreSQL + Auth + Realtime)
- **部署**: Vercel (前端) + Supabase Cloud (后端)
- **邮件**: Resend / Supabase Email

## ⏱ 开发周期

**总计**: 7天（每天6-8小时）

- Day 1: 项目初始化 + 数据库设计
- Day 2: 认证系统 + 基础布局
- Day 3: Profile引导流程
- Day 4: 名片池 + 标签筛选
- Day 5: 兴趣表达 + 双向匹配
- Day 6: 已连接页面 + Profile查看
- Day 7: 测试 + 优化 + 部署

## 🎯 MVP成功标准

1. ✅ 用户能完整走通注册→匹配→连接流程
2. ✅ 核心功能无阻塞性bug
3. ✅ 移动端基本可用
4. ✅ 成功部署到生产环境
5. ✅ 至少3个真实用户完成匹配

## 🚨 风险与应对

| 风险 | 应对措施 |
|------|---------|
| 冷启动问题（用户太少） | 邀请奇绩创坛校友先内测 |
| 匹配率低 | 初期人工协助匹配 |
| 技术债务 | MVP阶段接受一定技术债，V1.1重构 |
| 隐私安全问题 | 严格遵守Supabase RLS规则 |
